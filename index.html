Shana's Learning Plan

What was difficult last semester?

What happened: Not enough hands‑on coding practice

We didn’t have many opportunities to code along with the instructor in class. We covered the theory behind HTML and CSS elements, but we didn’t really dive into how to structure a webpage or explore the different ways to style it.

Why it was difficult: Layout and responsive design were confusing

I want more practice with structuring layouts using Grid and Flexbox, and I want to better understand when to use each one in different scenarios. Media queries were also challenging for me. I found it difficult to plan and organize styles for different screen sizes, such as creating four columns on desktop and one column on mobile, or replacing a navigation bar with a hamburger menu. Thinking through how to make a layout work across many screen sizes is still hard.

What will change: Build full projects and learn real workflow habits

I would also like more opportunities to build full projects so we can encounter real issues and learn how to solve them. In addition, I want to learn more about real-world coding practices used in workplaces.

My action plan for learning

How I’m Leveling Up My Coding Skills

  • Build full mini-projects to reinforce concepts through repetition.
  • Practice recreating exisitng websites layouts using Grid or Flexbox.
  • Follow a mobile-first approach and create a checklist for responsive design.

Weekly Coding Routine

  • Code 3 times per week, 45 minutes each session.
  • Tools: VS Code, MDN Web Docs, W3Schools, HTML and CSS Validator.
  • Checkpoint: At the end of each week, review what I built and write a short reflection and what could be improved.

Layout Practice Using Real Websites

  • Pick one website per week and recreate a section (header, hero, card grid, footer).
  • Build it twice: once with Flexbox, once with Grid.
  • Compare which method felt more efficient.
  • Tools: VS Code, Chrome DevTools, HTML and CSS Validator.
  • Checkpoint: Save screenshots and notes in a folder to track progress.

Responsive Design Practice

  • Create a personal checklist: breakpoints, typography scaling, spacing, navigation changes.
  • Build one responsive component per week (e.g., card grid, navbar, gallery).
  • Test on mobile, tablet, and desktop using DevTools.
  • Tools: VS Code, Chrome DevTools device simulator, MDN responsive design guide.
  • Checkpoint: By Week 4, complete a small responsive landing page.

One Full Mini‑Project Every Month

  • Build a 1–2 page website each month (portfolio section, product page, blog layout).
  • Include: semantic HTML, Flexbox/Grid, responsive design, and a simple navigation.
  • Tools: GitHub for version control, VS Code, HTML and CSS Validator.
  • Checkpoint: Share the project with a classmate or instructor for feedback.

Learn Real Workplace Practices

  • Follow a Git workflow: branch → commit → pull request.
  • Read one article per week on best practices (BEM, accessibility, naming conventions).
  • Tools: GitHub, CSS Tricks, Smashing Magazine.
  • Checkpoint: After 3 weeks, review what practices I’ve adopted and what still feels unclear.
